- int pollHighest() Removes and returns the eventId of the active event with the highest priority. If multiple active events have the same priority, return the smallest eventId among them. If there are no active events, return -1.

An event is called active if it has not been removed by pollHighest().

 

Example 1:

Input:
["EventManager", "pollHighest", "updatePriority", "pollHighest", "pollHighest"]

- len(a) = 3 and len(b) = 2

- median(a) = 3 and median(b) = 2

- cost = len(a) + len(b) + abs(median(a) - median(b)) = 3 + 2 + abs(3 - 2) = 6

Merge a = [1, 2, 3, 4, 5] and b = [6, 7, 8]:

- len(a) = 5 and len(b) = 3

- median(a) = 3 and median(b) = 7

- cost = len(a) + len(b) + abs(median(a) - median(b)) = 5 + 3 + abs(3 - 7) = 12

So lists becomes [[1, 2, 3, 4, 5, 6, 7, 8]], and total cost is 6 + 12 = 18.

Explanation:

Merge a = [1, 1, 5] and b = [1, 4, 7, 8]:

- len(a) = 3 and len(b) = 4

- median(a) = 1 and median(b) = 4

- cost = len(a) + len(b) + abs(median(a) - median(b)) = 3 + 4 + abs(1 - 4) = 10

There are no '1' characters in s, so no swaps can be performed. The score remains 0.

- 1 <= nums[i] <= 109

- s[i] is either '0' or '1'
